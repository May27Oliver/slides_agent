# Quickstart / 手動驗證: 015 編輯頁匯出與文字樣式覆寫

**Branch**: `015-editor-export-text-style`

針對無法（或不值得）自動化的行為的人工驗證路徑。對應 spec 的 Manual Verification Path 與 CR-015。

## 前置

1. 本機起前後端（沿用既有開發指令），登入有權限帳號。
2. 準備一個已生成、含**多張投影片 + 至少一張圖表 + 多條 outline** 的 deck，進其編輯頁。

## US1 — 下載 HTML

1. 不做任何編輯（dirty=false）→ 點「下載 HTML」。
2. 預期：下載 `<deck標題>-rev<N>-<時間戳>.html`；雙擊離線開啟，逐頁與線上預覽一致。
3. 改一個標題（dirty=true）→ 預期下載入口停用並提示先 Save。Save 後 → 可下載，內容為新 rev。

## US4 — 左側預覽 16:9

1. 改變瀏覽器視窗寬度（窄→寬）。預期：左側 slide 始終 16:9，多餘空間留邊（letterbox/pillarbox），內容不被壓扁/拉長。
2. 按 `F` 全螢幕。預期：仍維持 16:9 不變形。

## US3 — 文字大小與顏色覆寫

1. 點標題欄位的樣式控制 → 選 `XL` + `強調`。預期：左側預覽該頁標題即時放大並變強調色。
2. 對某一條 outline 設 `L` + `次要`。拖曳重排該條到別處 → 預期：樣式跟著該條移動，不錯位到別條。
3. 編輯該條文字（改字）→ 預期：樣式保留（id 不變）。
4. 刪除該條 → 預期：其樣式一併消失，其他條不受影響。
5. 對標題點「整欄 reset」→ 回主題預設大小與顏色。對另一欄只清大小（保留顏色）→ 驗證單屬性 reset。
6. Save → 重新整理頁面 → 預期：所有覆寫保留。
7. 下載 HTML（US1）開啟 → 預期：覆寫在離線檔中一致呈現。
8. 換一個主題（011）→ 預期：顏色覆寫（token）跟隨新主題的對應角色色，仍協調。

## US2 — 下載 PPTX

1. dirty=false 下點「下載 PPTX」→ 預期：顯示工作進行中（queued→processing），不阻塞編輯。
2. 等完成（done）→ 下載 `.pptx`，用 PowerPoint/Keynote 開啟。
   - 頁數 = 投影片數；每頁為對應 slide 的 16:9 滿版截圖；
   - 主題、圖表、**文字樣式覆寫**皆與線上一致（截圖涵蓋）。
3. **失敗/逾時路徑**：對極大 deck（>60 頁）→ 預期建立即被拒（400）。模擬 worker 失敗 → 預期狀態 failed、可重試、無殘檔可下載。
4. **隔離**：以另一帳號嘗試該 jobId 的狀態/下載 URL → 預期 404。

## 正式機（EC2 / docker-compose）部署驗證（PPTX 高風險項）

1. worker image 內含 chromium 與系統相依（research R2）。
2. 在正式機實際對一個 deck 跑一次 PPTX 匯出至 done 並下載開啟，確認無頭瀏覽器在容器內可正常截圖。
3. 確認 artifact TTL 到期後清理、暫存目錄無殘留。
